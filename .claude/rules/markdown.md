# Markdown Formatting Rules

When writing or editing markdown files, follow these formatting conventions:

## Headings

- All headings must be surrounded by blank lines (both before and after)
- This applies to all heading levels (H1-H6)
- Exception: The first heading in a document doesn't need a blank line before it

## Lists

- All lists must be surrounded by blank lines (both before and after)
- This includes both ordered and unordered lists
- This separates lists from other content and improves readability

## Code Blocks

- Use fenced code blocks (```) for multi-line code
- Specify the language after the opening ``` for syntax highlighting
- Separate code blocks from surrounding text with blank lines

## Emphasis

- Use `**bold**` for strong emphasis
- Use `*italic*` for emphasis
- Be consistent with emphasis style throughout the document

## Links

- Use reference-style links for repeated URLs: `[text][reference]`
- Place link references at the end of the document
- For external links, use the full URL format

## Line Length

- Prefer keeping lines under 80-100 characters when practical
- Hard-wrap only where it improves readability
- Don't break inline links or code elements

## Whitespace

- Use spaces for indentation (not tabs)
- Use 2 spaces for list indentation
- Use 4 spaces for code block indentation within fenced blocks
- No trailing whitespace at the end of lines

## Japanese Text

- For Japanese learning materials, preserve Japanese quotation marks 「」
- Maintain proper spacing around Japanese text when mixed with English
- Keep furigana and reading aids as originally formatted
